## 性能优化 / Performance

### PR #26 — MCP 工具调用延迟降低（6 项优化）

| # | 优化内容 | 文件 | 单次节省 |
|---|---|---|---|
| 1 | Probe 已就绪时跳过 `await`（快速路径） | `debug_tools_native.gd` | ~16ms |
| 2 | 批量工具日志写入（内存缓冲，每 20 次刷盘） | `mcp_server_core.gd` | ~10ms |
| 3 | HTTP 服务器循环延迟 10ms → 2ms | `mcp_http_server.gd` | ~8ms |
| 4 | 截图默认 JPG（编码速度比 PNG 快 5-10 倍） | `editor_tools_native.gd`、`debug_tools_native.gd` | ~250ms |
| 5 | 更轻量的请求去重 key（str 拼接替代 JSON.stringify） | `debug_tools_native.gd` | ~1ms |
| 6 | 服务器停止时刷盘日志缓冲 | `mcp_server_core.gd` | 正确性 |

**关键 Bug 修复 — 重复运行时表达式 35.9 倍加速**

`_extract_pending_runtime_probe_response` 中急切的 stale 回退（`get_latest_message_payload`）在旧匹配负载存在时立即返回 `"stale"`，`_request_runtime_probe_poll` 将 `"stale"` 视为可重试，导致重复表达式在首次尝试后返回 stale 并燃尽完整 3 秒超时。

测量结果（1+1 表达式，8 次连续调用）：
- 修复前：平均 3.056s 每次（stale, from_cache）
- 修复后：平均 0.085s 每次（fresh）→ **35.9 倍加速**
