## 🆕 新功能 / New Features

- **154 个 MCP 工具全量实现**：涵盖节点操作、脚本管理、场景控制、编辑器交互、项目管理、运行时调试、动画控制、视觉属性、输入模拟等完整能力
  **154 MCP tools fully implemented**: Covering node operations, script management, scene control, editor interaction, project management, runtime debugging, animation control, visual properties, input simulation, and more.

- **工具管理面板**：支持按分组动态启用/禁用工具，核心工具（30个）默认启用，补充工具（124个）按需开启
  **Tool Management Panel**: Dynamically enable/disable tools by group. 30 core tools enabled by default, 124 supplementary tools opt-in.

- **内置多语言支持**：MCP 面板支持中英文切换，设置参数持久化
  **Built-in i18n Support**: MCP panel supports Chinese/English switching with persistent settings.

- **Vibe Coding 免打扰模式**：默认启用，避免 UI 操作打断编码流程
  **Vibe Coding Quiet Mode**: Enabled by default, prevents UI operations from interrupting coding flow.

- **Debugger Bridge + Runtime Probe**：运行时调试基础设施，支持断点、变量检查、表达式求值、性能监控
  **Debugger Bridge + Runtime Probe**: Runtime debugging infrastructure with breakpoints, variable inspection, expression evaluation, and performance monitoring.

- **运行时动画控制**：AnimationPlayer 播放/停止/状态查询，AnimationTree 状态机 travel/active 控制
  **Runtime Animation Control**: AnimationPlayer play/stop/state query, AnimationTree state machine travel/active control.

- **运行时视觉属性**：Material/Theme/Shader/TileMap 运行时读写
  **Runtime Visual Properties**: Runtime read/write for Material, Theme, Shader, and TileMap.

- **运行时输入模拟**：键盘、鼠标、Action 等结构化输入事件
  **Runtime Input Simulation**: Structured input events for keyboard, mouse, and actions.

- **PR 审查与合并流程**：集成分支审查、GUT 全量测试、Squash Merge 合并
  **PR Review & Merge Workflow**: Integration branch review, full GUT test suite, squash merge.

- **多 AI 客户端配置**：支持 Cline、OpenCode、Claude Code、Codex 配置格式
  **Multi AI Client Configs**: Support for Cline, OpenCode, Claude Code, and Codex configuration formats.

## 🐛 Bug 修复 / Bug Fixes

- **修复端口占用问题**：增加报错提示，避免静默失败
  **Fix port conflict**: Added error message instead of silent failure.

- **修复 AnimationTree 序列化 `float()` 构造函数错误**：Godot 4.6 不支持 `float()` 构造函数，改为 `as float` 类型转换（影响 5 处）
  **Fix `float()` constructor error in AnimationTree serialization**: Godot 4.6 does not support `float()` constructor; replaced with `as float` type cast (5 occurrences).

- **修复 `travel_runtime_animation_tree` 的 `match_fields` 过滤 bug**：移除异步更新的 `current_node` 字段，避免响应被跳过
  **Fix `match_fields` filtering bug in `travel_runtime_animation_tree`**: Removed asynchronously-updated `current_node` field to prevent response from being skipped.

## ⚠️ 兼容性说明 / Compatibility Notes

- **Godot 4.6+**：`float()` 构造函数不存在，需用 `as float`
  **Godot 4.6+**: `float()` constructor does not exist; use `as float` instead.

- **Godot 4.x**：`AnimationNodeStateMachine.set_start_node()` 不存在，用 `add_node()` 替代
  **Godot 4.x**: `AnimationNodeStateMachine.set_start_node()` does not exist; use `add_node()` instead (first added node becomes the start node).

- **Runtime Probe TileMap 工具**：仅支持旧式 `TileMap` 节点，不支持 Godot 4.3+ 的 `TileMapLayer`
  **Runtime Probe TileMap tools**: Only support legacy `TileMap` node; Godot 4.3+ `TileMapLayer` is not supported.
